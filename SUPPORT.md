# Support

## Where to Ask What

Use the right channel for the right type of issue:

- **Bug reports**: GitHub Issues
- **Feature requests**: GitHub Issues or Discussions if enabled
- **Security vulnerabilities**: email only, see [SECURITY.md](SECURITY.md)
- **Code of Conduct concerns**: see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
- **Operational questions about self-hosting**: GitHub Issues, with deployment details
- **Source-quality or provenance concerns**: GitHub Issues, clearly labeled

## When Opening a Bug Report

Include:

- what you expected to happen
- what actually happened
- exact route, component, or command if known
- whether you are running locally, in Docker, or on a VPS
- relevant logs or screenshots with secrets removed
- whether the problem affects ingestion, UI, auth, exports, or deployment

## When Reporting a Data Problem

For source-quality issues, include:

- source name
- approximate timestamp
- region or AOI affected
- whether the problem is bad geocoding, stale data, duplicate signals, attribution, or confidence labeling

## Response Expectations

This is an open-source project and support is best-effort. Clear, reproducible, well-scoped reports are much easier to triage and fix quickly.
